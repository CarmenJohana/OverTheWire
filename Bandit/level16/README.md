# Bandit Level 16 — Scanning SSL/TLS Services

## Overview

In **Bandit Level 16**, the objective is to find a service running on a port
between **31000 and 32000** that uses **SSL/TLS**, submit the current password,
and retrieve credentials for the next level.

This level focuses on:
- Port scanning
- Identifying SSL/TLS-enabled services
- Distinguishing between similar-looking services

---

## Port Scanning

To identify open ports and services, `nmap` was used.  
The scan revealed the following open ports:


	PORT      STATE SERVICE     VERSION
	31046/tcp open  echo
	31518/tcp open  ssl/echo
	31691/tcp open  echo
	31790/tcp open  ssl/unknown
	31960/tcp open  echo

## Why ssl/echo Does NOT Work

At first glance, _ssl/echo_ (port 31518) seems like the correct choice.
However, it does not solve the level.

### Explanation

- An echo service simply sends back whatever data you send to it

- ssl/echo means:

	- The connection is encrypted

	- BUT the service does not perform any validation or logic

- When the password is sent, the service just echoes it back

- No credentials or private key are returned

In contrast:

- ssl/unknown indicates:

	- A TLS-encrypted service

	- With custom server-side logic

- This is the only service capable of verifying the password and returning
the private key required for the next level

## Solution

The correct port is:

	31790/tcp — ssl/unknown

Using _ncat_ with SSL support:

	ncat localhost --ssl 31790

After connecting, the password for the current level was submitted:

	kSkvUpMQ7lBYyCM4GBPvCvT1BfWRy0Dx
